- Fixed all 8 `tenant.id` -> `tenant.tenant_id` call sites across estimation_service.py (2) and feedback_service.py (6), eliminating AttributeError that would fire on every get_estimation, upload_image, and feedback operation

- Removed POST /estimate/start route from routes.py, removed EstimationRequest import, removed 5 legacy handler methods (start_estimation, _process_estimation, _generate_estimation, _parse_estimation_response, _create_default_estimation) and 2 unused imports (EstimationRequest, EstimationResult) from estimation_service.py

## Decisions Made

- `tenant.tenant_id` is the correct field -- Tenant model has `tenant_id: str` as primary identifier, no `.id` attribute
- `connect_to_mongo()` and `create_indexes()` outside any try/except in lifespan -- both must crash app on failure (fail-fast), matching PromptService pattern already in place
- `rotate_openai_key()` was a thin one-line wrapper calling `validate_and_store_openai_key()` with no external callers -- deletion is safe and reduces confusion
- `get_tenant_allowed_origins()` was superseded by the CORS cache lazy-population inside `get_branding_by_prefix()` -- no separate callers needed the standalone function
