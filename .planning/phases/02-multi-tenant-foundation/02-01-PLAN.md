**Create `app/services/auth_service.py`** — New file (replace existing skeleton in security.py's AuthService — that class will be updated in plan 02-02). Functions:

1. `register_tenant(body: RegisterRequest) -> RegisterResponse`:
   - Check email not already registered. If duplicate: return 201 with generic message "If this email is new, check your inbox for verification." (prevent email enumeration per locked decision)
   - Generate `tenant_id = str(uuid.uuid4())`
   - Hash password with `pwdlib.PasswordHash.recommended().hash(body.password)`
   - Generate API key: `raw_api_key = f"sk_live_{tenant_id.replace('-', '')}_{secrets.token_urlsafe(16)}"` — hash with pwdlib for storage
   - Build tenant_doc with all Tenant fields, `tier="trial"`, `email_verified=False`
   - Insert into tenants collection (direct db access, NOT TenantAwareCollection — tenants collection is not tenant-scoped)
   - Generate verification token and store in verification_tokens collection
   - Send verification email (background task — use FastAPI BackgroundTasks)
   - Return RegisterResponse with tenant_id, raw_api_key (shown once), and message

2. `verify_email(token: str) -> VerifyEmailResponse`:
   - Look up token in verification_tokens collection
   - If not found or expired: raise 400 "Invalid or expired verification token"
   - Set tenant's `email_verified = True`
   - Delete the used token (single-use)
   - Return success response

3. `send_verification_email(email: str, token: str)`:
   - Use fastapi-mail ConnectionConfig from settings
   - Build verification URL: `{settings.APP_BASE_URL}/api/v1/auth/verify?token={token}`
   - Send plain-text email with link and 24-hour expiry notice
   - If SMTP not configured (SMTP_USERNAME is None), log warning and skip (dev mode)

4. `generate_verification_token() -> tuple[str, datetime]`:
   - `secrets.token_urlsafe(32)` for token
   - Expiry: `datetime.now(timezone.utc) + timedelta(hours=24)`

5. `update_profile(tenant_id: str, body: ProfileUpdateRequest) -> ProfileResponse`:
   - Fetch tenant by tenant_id
   - Update allowed fields (company_name, settings)
   - Return ProfileResponse with masked API key (`sk-...{last6}`)

6. `get_profile(tenant_id: str) -> ProfileResponse`:
   - Fetch tenant, return ProfileResponse
   - `has_openai_key = tenant["encrypted_openai_key"] is not None`
   - `masked_api_key = "sk-...{last6}"` (cannot recover from hash — store last 6 chars alongside hash)

**Important detail for API key masking:** Since the API key is bcrypt-hashed, we cannot reverse it to show last 6 chars later. Store `api_key_last6: str` alongside `hashed_api_key` in the tenant document. Update the Tenant model accordingly (add `api_key_last6: str = ""`).
