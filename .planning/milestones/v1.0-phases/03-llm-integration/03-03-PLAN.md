<objective>
Rewrite the ChatService as a real conversation state machine that guides users through structured project scoping with LLM-powered follow-up questions, tracks scoping context, detects readiness, and persists sessions in MongoDB with TTL auto-expiry.

Purpose: The existing ChatService is a stub with hardcoded keyword matching. This plan replaces it with a production conversation engine that: (1) uses the scoping prompt to generate contextual follow-up questions, (2) tracks which project details have been gathered via a ScopingContext model, (3) uses rule-based readiness detection for speed/cost, and (4) stores full conversation history in MongoDB with TTL. This is the interactive layer that feeds into narrative generation (plan 03-04).

Output: A fully functional conversational scoping engine that feels like talking to a knowledgeable contractor.
</objective>

   ```python
   class ScopingContext(BaseModel):
       """Tracks which project details have been gathered during scoping."""
       project_type: Optional[str] = Field(None, description="Type of project (e.g., pool, deck, renovation)")
       project_size: Optional[str] = Field(None, description="Approximate size/scope (e.g., '15x30 feet', '500 sqft')")
       location: Optional[str] = Field(None, description="Project location/region")
       timeline: Optional[str] = Field(None, description="Desired timeline (e.g., 'spring 2026', '3 months')")
       special_conditions: Optional[str] = Field(None, description="Special conditions (access, soil, HOA, etc.)")

       def populated_fields(self) -> set[str]:
           """Return set of field names that have non-None values."""
           return {k for k, v in self.model_dump().items() if v is not None}

       def is_ready(self) -> bool:
           """True when enough detail exists for a quality estimate.
           Required: project_type, location. Sufficient: project_type, project_size, location, timeline.
           """
           populated = self.populated_fields()
           return {"project_type", "project_size", "location", "timeline"}.issubset(populated)

       def missing_fields(self) -> list[str]:
           """Return list of field names that are still None, in priority order."""
           priority = ["project_type", "project_size", "location", "timeline", "special_conditions"]
           return [f for f in priority if getattr(self, f) is None]
   ```

**Rewrite `app/services/chat_service.py`:**

Full replacement. The new ChatService:

1. **Constructor** — accepts `llm_service: LLMService` (injected, not self-created):
   ```python
   class ChatService:
       ESTIMATE_TRIGGER_PHRASES = {"generate estimate", "give me an estimate", "ready for estimate", "create estimate", "/estimate"}

       def __init__(self, llm_service: LLMService):
           self.llm_service = llm_service
   ```

2. **`send_message(self, request: ChatRequest, tenant: Tenant) -> ChatResponse`:**
   - Get or create session via `_get_or_create_session(request.session_id, tenant)`
   - Append user message to session.messages: `ChatMessage(role="user", content=request.message)`
   - Check if user explicitly triggered estimate: `_is_explicit_estimate_trigger(request.message)`
   - If explicit trigger or user confirmed readiness (e.g., "yes", "ready", "go"):
     - If `session.is_ready` or explicit trigger: set `session.status = "ready"` and return confirmation response
     - If not ready, respond saying more info is needed
   - Otherwise: call `_generate_follow_up(session)` to get LLM response
   - After LLM response, extract context updates: `_update_scoping_context(session, request.message, llm_response)`
   - Check readiness: `session.scoping_context.is_ready()`
   - If newly ready: append auto-trigger message: "I have enough details to generate an estimate for your project. Shall I go ahead?"
   - Append assistant message to session.messages
   - Persist session to MongoDB
   - Return ChatResponse with current state

3. **`_get_or_create_session(self, session_id: Optional[str], tenant: Tenant) -> ChatSession`:**
   - If session_id provided: look up in DB using `get_tenant_collection`
   - If found: deserialize to ChatSession and return
   - If not found or no session_id: create new session with `session_id = f"chat_{uuid.uuid4().hex[:12]}"`
   - Set `prompt_version = PromptService.get_version_string("scoping", "latest")`
   - Insert into DB and return

4. **`_generate_follow_up(self, session: ChatSession) -> str`:**
   - Load scoping prompt: `prompt = PromptService.get("scoping", session.prompt_version or "latest")`
   - Build conversation history string from session.messages (last 10 messages for context window)
   - Build scoping context string from session.scoping_context
   - Format user_prompt_template with: `conversation_history`, `user_message` (last user message), `scoping_context`
   - Call `self.llm_service.generate_response(formatted_user_prompt, prompt["system_prompt"])`
   - Return the response string

5. **`_update_scoping_context(self, session: ChatSession, user_message: str, llm_response: str)`:**
   - Use a simple LLM call with a structured extraction prompt to extract context from the user's message.
   - Alternative (simpler, recommended): Use keyword/pattern extraction similar to the existing approach but improved:
     - Project type: check for keywords (pool, deck, patio, renovation, addition, kitchen, bathroom, roof, fence, landscaping, etc.)
     - Size: regex for dimensions like `\d+\s*x\s*\d+`, `\d+\s*sq\s*ft`, `\d+\s*square\s*feet`
     - Location: check for known region names from `Region` enum, or general location patterns
     - Timeline: check for month names, "spring/summer/fall/winter", "ASAP", time references
     - Special conditions: check for keywords (HOA, slope, access, soil, permits, etc.)
   - Only update fields that are currently None (don't overwrite existing context)
   - This is a "good enough" extraction — the LLM follow-up questions will naturally gather the info we need. The context extraction is a bonus for readiness detection.

6. **`_is_explicit_estimate_trigger(self, message: str) -> bool`:**
   - Check if the lowercased, stripped message matches any phrase in `ESTIMATE_TRIGGER_PHRASES`
   - Also check for partial matches: `any(trigger in message.lower() for trigger in ESTIMATE_TRIGGER_PHRASES)`

7. **`_is_confirmation(self, message: str) -> bool`:**
   - Check if message is a short affirmative: "yes", "yeah", "sure", "go ahead", "ready", "go", "yep", "do it"
   - Used when `session.is_ready` is True and user is confirming estimate generation

8. **`get_chat_history(self, session_id: str, tenant: Tenant) -> list[dict]`:**
   - Fetch session from DB
   - Return `[msg.model_dump() for msg in session.messages]`

9. **`_persist_session(self, session: ChatSession, tenant: Tenant)`:**
   - Use `get_tenant_collection(DB_COLLECTIONS["CHAT_SESSIONS"], tenant.tenant_id)`
   - Upsert: `update_one({"session_id": session.session_id}, {"$set": session.model_dump(by_alias=True)}, upsert=True)`
   - Update `session.updated_at` before persisting

**Error handling in `send_message`:**
- Wrap the LLM call in try/except for OpenAIError
- On transient error: preserve conversation, return error message: "We're having trouble generating a response. Retrying..." (per locked decision: transparent with retry)
- On auth/quota error: return 402-appropriate message
- Conversation is ALWAYS preserved in DB before returning error — user never loses progress

```python
@api_router.post("/chat/send", response_model=ChatResponse)
@limiter.limit(get_tier_limit, key_func=get_tenant_id_for_limit)
async def send_chat_message(
    http_request: Request,
    request: ChatRequest,
    tenant: Tenant = Depends(get_current_tenant),
    chat_service: ChatService = Depends(get_chat_service)
):
    """Send a chat message for conversational project scoping."""
    try:
        response = await chat_service.send_message(request, tenant)
        return response
    except HTTPException:
        raise  # Let 402 etc. propagate
    except Exception as e:
        logger.error(f"Error sending chat message: {e}")
        raise HTTPException(
            status_code=HTTP_STATUS["INTERNAL_ERROR"],
            detail="Failed to process chat message"
        )
```
