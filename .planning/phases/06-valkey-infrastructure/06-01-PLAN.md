<objective>
Replace the per-process `_response_cache` dict in `llm_service.py` with a distributed `ValkeyCache` service backed by Valkey. After this plan, LLM response caching works across all Gunicorn workers, cache keys are tenant-scoped (`efofx:llm:{tenant_id}:{input_hash}`), and Valkey outages fall back silently to live LLM calls.

Purpose: The current per-process dict cache is invisible across Gunicorn workers — Worker A caches a result that Worker B never sees. This makes caching effectively useless in production. Distributed Valkey cache fixes this and adds tenant isolation.

Output: `ValkeyCache` service class, wired into `LLMService.generate_estimation()`, with graceful fallback, lifespan cleanup, and comprehensive unit tests.
</objective>

**Create `apps/efofx-estimate/app/services/valkey_cache.py`:**

Create a `ValkeyCache` class with:

1. **Constructor** `__init__(self)`: sets `self._client: Optional[valkey.Valkey] = None`. No connection at init (lazy).

2. **`_get_client(self) -> valkey.Valkey`**: Lazy-initializes the async Valkey client using `valkey.asyncio.Valkey.from_url(settings.VALKEY_URL, decode_responses=True, socket_timeout=2.0, socket_connect_timeout=2.0)`. Returns the cached client on subsequent calls. Import as `import valkey.asyncio as valkey_async` to avoid shadowing.

3. **`_make_key(self, tenant_id: str, input_hash: str) -> str`**: Returns `f"efofx:llm:{tenant_id}:{input_hash}"`. This is the locked key format from CONTEXT.md.

4. **`make_input_hash(messages: list[dict], model: str) -> str`** (static method): SHA-256 hash of `json.dumps({"messages": messages, "model": model}, sort_keys=True, ensure_ascii=True)`. This replaces the standalone `_make_cache_key` function — same algorithm, same output.

5. **`_maybe_warn(self, exc: Exception) -> None`**: Uses module-level `_last_warn_at: float = 0.0` and `_WARN_COOLDOWN_SECONDS = 60`. Only logs `logger.warning("Valkey unavailable — falling back to live LLM call: %s", exc)` if `time.monotonic() - _last_warn_at > _WARN_COOLDOWN_SECONDS`. Updates `_last_warn_at` after logging.

6. **`async get(self, tenant_id: str, input_hash: str) -> Optional[str]`**: Try `await client.get(key)`. On hit, log debug "Valkey cache HIT". On miss, log debug "Valkey cache MISS". Catch `(ValkeyConnectionError, ValkeyTimeoutError)` -> `_maybe_warn()` -> return `None`.

7. **`async set(self, tenant_id: str, input_hash: str, value: str) -> None`**: Try `await client.set(key, value, ex=settings.VALKEY_CACHE_TTL)`. Catch same exceptions -> `_maybe_warn()` -> silently no-op.

8. **`async close(self) -> None`**: If `self._client is not None`, call `await self._client.aclose()` then set `self._client = None`.

9. **Module-level singleton**: `_cache = ValkeyCache()` at bottom of file.

Import exceptions as:
```python
from valkey.exceptions import ConnectionError as ValkeyConnectionError, TimeoutError as ValkeyTimeoutError
```
