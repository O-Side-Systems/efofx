**Create `app/services/prompt_service.py`:**

```python
"""
Prompt registry for versioned, immutable prompt management.

Prompts are stored as JSON files in config/prompts/ and loaded once at app startup.
Each prompt has a name (e.g., "scoping", "narrative", "estimation") and a semver
version string. Versions are immutable — any change to content requires a new version.

Usage:
    PromptService.load_all("config/prompts/")  # called in app lifespan
    prompt = PromptService.get("narrative", "1.0.0")
    prompt = PromptService.get("narrative", "latest")  # resolves to highest semver
"""
```

Class structure:
- `_registry: dict[str, dict]` — class variable, keyed by `"{name}:{version}"`
- `_content_hashes: dict[str, str]` — class variable, keyed by `"{name}:{version}"`, value is SHA-256 of file content
- `load_all(cls, prompts_dir: str) -> None` — classmethod
  - Iterate all `.json` files in `prompts_dir` using `pathlib.Path.glob("*.json")`
  - For each file: read text, compute SHA-256 hash, parse JSON
  - Validate required fields: `version`, `name`, `created_at`, `system_prompt`, `user_prompt_template`
  - Build registry key: `f"{data['name']}:{data['version']}"`
  - If key already exists and content hash differs: raise `ValueError(f"Immutability violation: {key} content changed")`
  - Store in `_registry` and `_content_hashes`
  - Log number of prompts loaded
- `get(cls, name: str, version: str = "latest") -> dict` — classmethod
  - If `version == "latest"`: find all entries matching `name:*`, sort by semver (use `packaging.version.Version` or simple tuple split), return highest
  - Otherwise: look up `f"{name}:{version}"` directly
  - Raise `KeyError(f"Prompt not found: {name}:{version}")` if missing
- `get_version_string(cls, name: str, version: str = "latest") -> str` — classmethod
  - Returns just the version string (e.g., "1.0.0") — convenience for populating `prompt_version` field
  - Calls `get()` internally and returns `result["version"]`
- `list_versions(cls, name: str) -> list[str]` — classmethod
  - Returns sorted list of all versions for a prompt name
- `clear(cls) -> None` — classmethod (for test isolation)
  - Resets `_registry` and `_content_hashes` to empty dicts

**Important:** Do NOT use `packaging` library for semver sorting. Use a simple approach:
```python
def _semver_key(version_str: str) -> tuple:
    return tuple(int(x) for x in version_str.split("."))
```

**Create prompt JSON files:**

Each file follows this schema:
```json
{
  "version": "1.0.0",
  "name": "<prompt_name>",
  "created_at": "2026-02-27",
  "description": "<what this prompt does>",
  "system_prompt": "<the system message>",
  "user_prompt_template": "<the user message template with {placeholders}>"
}
```

**`config/prompts/v1.0.0-scoping.json`:**
- name: "scoping"
- system_prompt: A system prompt that instructs the LLM to act as a knowledgeable, friendly contractor helping a homeowner scope their project. The LLM should:
  - Ask focused questions one at a time (not a list)
  - Cover: project type, approximate size/scope, location/region, desired timeline, any special conditions
  - Be conversational — feel like talking to a knowledgeable contractor, not filling out a form
  - When enough detail exists (project_type + size + location + timeline all present), indicate readiness
  - Use plain language, avoid construction jargon
- user_prompt_template: `"Conversation so far:\n{conversation_history}\n\nUser's latest message: {user_message}\n\nProject context gathered so far:\n{scoping_context}\n\nRespond with your next message. **`config/prompts/v1.0.0-narrative.json`:**
- name: "narrative"
- system_prompt: A system prompt that instructs the LLM to generate a plain-language estimate narrative for homeowners. The LLM should:
  - Lead with the bottom-line P50/P80 total range in plain language ("You'll most likely spend between $X-$Y")
  - Use "most likely" for P50 and "could reach" or "budget for" for P80
  - Explain each cost category in plain English — what it covers and why the range exists
  - Reference specific adjustment factors and explain their impact
  - Avoid statistical terminology — no "percentile", "confidence interval", etc.
  - Be specific to the project described, not generic boilerplate
  - Include timeline ranges in the same plain-language style
- user_prompt_template: `"Project description: {project_description}\n\nEstimate data:\n- Total cost (most likely): ${total_cost_p50:,.0f}\n- Total cost (budget for): ${total_cost_p80:,.0f}\n- Timeline (most likely): {timeline_weeks_p50} weeks\n- Timeline (budget for): {timeline_weeks_p80} weeks\n\nCost breakdown:\n{cost_breakdown}\n\nAdjustment factors:\n{adjustment_factors}\n\nAssumptions:\n{assumptions}\n\nGenerate a plain-language narrative explaining this estimate to a homeowner."`

**Modify `app/main.py`:**

Add PromptService loading to the lifespan startup:
```python
from app.services.prompt_service import PromptService
import os

@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup
    logger.info("Starting efOfX Estimation Service...")
    try:
        await connect_to_mongo()
        logger.info("MongoDB connection established")
    except Exception as e:
        logger.error(f"Failed to connect to MongoDB: {e}")

    # Load versioned prompts
    prompts_dir = os.path.join(os.path.dirname(os.path.dirname(__file__)), "config", "prompts")
    try:
        PromptService.load_all(prompts_dir)
        logger.info("Prompt registry loaded")
    except Exception as e:
        logger.error(f"Failed to load prompt registry: {e}")
        raise  # Prompts are critical — fail startup if they can't load

    yield

    # Shutdown
    logger.info("Shutting down efOfX Estimation Service...")
    await close_mongo_connection()
    logger.info("MongoDB connection closed")
```

**Important:** The `prompts_dir` path must resolve correctly from `app/main.py` to `config/prompts/`. The app structure is:
```
apps/efofx-estimate/
  app/
    main.py     <- __file__ is here
  config/
    prompts/    <- target directory
```
So: `os.path.join(os.path.dirname(os.path.dirname(os.path.abspath(__file__))), "config", "prompts")`
