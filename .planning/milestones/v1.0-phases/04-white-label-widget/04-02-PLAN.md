**Step 2: Create `app/services/widget_service.py`** — business logic for widget operations.

Three main functions:

`get_branding_by_prefix(api_key_prefix: str) -> Optional[BrandingConfigResponse]`:
- Takes the 32-char hex prefix (tenant_id without dashes)
- Reconstructs UUID format: `{0:8}-{8:12}-{12:16}-{16:20}-{20:32}` (same pattern as `_validate_api_key` in security.py)
- Queries `tenants` collection by `{"tenant_id": tid}`
- If tenant found: extract `tenant.settings.get("branding", {})`, merge with BrandingConfig defaults, set `company_name` from `tenant["company_name"]` if not overridden in branding settings
- Return `BrandingConfigResponse` with safe fields only
- If not found: return `None`

`save_lead(tenant_id: str, lead: LeadCaptureRequest) -> str`:
- Creates a LeadCapture document from the request + tenant_id
- Inserts into a tenant-scoped `widget_leads` collection (use `get_tenant_collection` pattern)
- Returns the inserted document ID as string

`record_analytics_event(tenant_id: str, event_type: str) -> None`:
- Gets today's date as `YYYY-MM-DD` string
- Upserts into `widget_analytics` collection using `$inc`:
  ```python
  filter = {"tenant_id": tenant_id, "date": today}
  update = {"$inc": {event_type: 1}}
  await collection.update_one(filter, update, upsert=True)
  ```
- Fire-and-forget pattern (caller does not await result in production)

Also add `get_tenant_allowed_origins(api_key_prefix: str) -> list[str]`:
- Same prefix-to-UUID lookup as branding
- Returns `tenant.settings.get("allowed_origins", [])` or empty list

**Step 1: Create `app/middleware/cors.py`** — TenantAwareCORSMiddleware.

Subclass Starlette's `CORSMiddleware` to check per-tenant allowed origins:

```python
from starlette.middleware.cors import CORSMiddleware
from starlette.types import ASGIApp
from typing import Sequence
import logging

logger = logging.getLogger(__name__)


class TenantAwareCORSMiddleware(CORSMiddleware):
    """Extends CORSMiddleware with per-tenant dynamic origin checking.

    First checks the static allow_origins list (admin dashboard, dev).
    Then checks tenant-registered domains stored in Tenant.settings['allowed_origins'].
    Uses an in-memory cache (dict) to avoid DB hits on every preflight.
    """

    def __init__(self, app: ASGIApp, *, allow_origins: Sequence[str] = (), **kwargs):
        super().__init__(app, allow_origins=allow_origins, **kwargs)
        # In-memory cache: tenant_id -> list of allowed origins
        # Populated by widget_service during branding lookup, refreshed on each lookup
        self._tenant_origins: dict[str, list[str]] = {}

    def is_allowed_origin(self, origin: str) -> bool:
        # Static origins first (fast path — admin/dashboard/dev)
        if super().is_allowed_origin(origin):
            return True
        # Check all cached tenant origins
        for origins in self._tenant_origins.values():
            if origin in origins:
                return True
        return False

    def update_tenant_origins(self, tenant_id: str, origins: list[str]) -> None:
        """Called by widget_service to update cached origins for a tenant."""
        self._tenant_origins[tenant_id] = origins
```

The cache is populated lazily: when the branding endpoint is called, the widget_service also updates the CORS cache. This avoids loading all tenant origins at startup and avoids async DB calls in middleware.

IMPORTANT: Starlette's `add_middleware` creates a new instance internally. To keep a reference, we need to either:
- Use `app.add_middleware(TenantAwareCORSMiddleware, ...)` and then access it via app middleware stack, OR
- Store the class on `app.state` and have `widget_service` access it via the request's `app.state`

The simplest approach: keep `app.add_middleware(TenantAwareCORSMiddleware, ...)` as before, and have the widget_service update origins via a module-level dict that the middleware reads. Create a module-level `_tenant_origins_cache: dict[str, list[str]] = {}` in `cors.py` that both the middleware class and widget_service can import.
