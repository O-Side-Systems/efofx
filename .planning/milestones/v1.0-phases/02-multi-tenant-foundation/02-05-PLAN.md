**Create `app/core/rate_limit.py`:**

```python
"""Rate limiting with slowapi and Valkey backend."""
from slowapi import Limiter
from slowapi.errors import RateLimitExceeded
from slowapi.util import get_remote_address
from fastapi import Request
from fastapi.responses import JSONResponse
from typing import Optional

from app.core.config import settings

# Tier-based rate limits (Claude's discretion per CONTEXT.md)
TIER_LIMITS = {
    "trial": "20/minute",
    "paid": "100/minute",
}

# Default limit for unauthenticated endpoints
DEFAULT_LIMIT = "30/minute"


def get_tenant_id_for_limit(request: Request) -> str:
    """Key function for per-tenant rate limiting.

    Extracts tenant_id from request.state (set by JWT middleware).
    Falls back to IP-based limiting for unauthenticated requests.
    """
    tenant_id = getattr(request.state, "tenant_id", None)
    if tenant_id:
        return f"tenant:{tenant_id}"
    return f"ip:{get_remote_address(request)}"


def get_dynamic_limit(key: str) -> str:
    """Return rate limit string based on tenant tier.

    This is called by slowapi to determine the limit for each request.
    The request object isn't directly available here, so we use a middleware
    approach: set request.state.rate_limit before the limiter runs.

    For simplicity, use a fixed approach: apply the limiter with a dynamic
    limit string function on per-route basis.
    """
    return DEFAULT_LIMIT


# Create limiter with Valkey backend
# Uses VALKEY_URL from settings. In dev: redis://localhost:6379 (plain)
# In production: valkeys://user:pass@host:25061 (SSL)
# Note: slowapi/limits library uses "redis" scheme even for Valkey (same protocol)
limiter = Limiter(
    key_func=get_tenant_id_for_limit,
    default_limits=[DEFAULT_LIMIT],
    storage_uri=settings.VALKEY_URL,
    enabled=settings.RATE_LIMIT_ENABLED,
)


async def rate_limit_exceeded_handler(
    request: Request, exc: RateLimitExceeded
) -> JSONResponse:
    """Custom 429 handler returning standardized JSON body.

    Per locked decision:
    - 429 response includes: {"error": "rate_limit_exceeded", "message": "...", "retry_after": N}
    - Retry-After header included
    - Rate limit headers: X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset
    """
    # Extract retry_after from the exception detail string
    # slowapi format: "Rate limit exceeded: X per Y"
    retry_after = 60  # default fallback
    detail = str(exc.detail) if hasattr(exc, "detail") else "Rate limit exceeded"

    response = JSONResponse(
        status_code=429,
        content={
            "error": "rate_limit_exceeded",
            "message": f"Rate limit exceeded. {detail}",
            "retry_after": retry_after,
        },
        headers={
            "Retry-After": str(retry_after),
        },
    )
    return response
```

**Important implementation notes:**
- slowapi automatically adds `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset` headers to responses when the limiter is active. No manual header injection needed.
- For per-tenant tier-based limits, use `@limiter.limit(dynamic_limits_func)` pattern where the function reads the tenant's tier from request.state.
- The `storage_uri` connects to Valkey using the same Redis protocol. In dev, use `redis://localhost:6379`. In production, use the Valkey URL from DigitalOcean.
- If Valkey is not available (dev without Docker), slowapi falls back to in-memory storage (acceptable for local dev, not for production).

**Update `app/api/routes.py`** — Apply per-tenant rate limiting to existing API routes:

For each protected endpoint (those using `Depends(get_current_tenant)`), apply the tenant-based limiter. The approach:

1. Add a middleware or dependency that sets `request.state.tenant_id` and `request.state.tier` from the JWT token. This allows the rate limiter key function to use tenant context.

2. Apply `@limiter.limit(dynamic_limit_string)` to protected routes. For dynamic per-tier limits, use a function:
```python
def get_tier_limit(request: Request) -> str:
    """Return limit string based on tenant tier from request state."""
    tier = getattr(request.state, "tier", "trial")
    return TIER_LIMITS.get(tier, TIER_LIMITS["trial"])
```

Then on routes: `@limiter.limit(get_tier_limit, key_func=get_tenant_id_for_limit)`

Alternatively, use slowapi's `@limiter.shared_limit()` pattern for shared limits across routes.

**Important:** If the existing routes in routes.py don't currently use `request: Request` as a parameter, add it. slowapi requires it.
