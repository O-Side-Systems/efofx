2. **Create** `packages/efofx-shared/efofx_shared/core/constants.py` with ONLY the four pure enum classes from `apps/efofx-estimate/app/core/constants.py`:

3. **Update** `apps/efofx-estimate/app/core/constants.py`: Remove the four enum class definitions and replace with imports from the shared package:
   ```python
   from efofx_shared.core.constants import (
       EstimationStatus,
       ReferenceClassCategory,
       CostBreakdownCategory,
       Region,
   )
   ```
   Keep all estimation-specific constants (API_MESSAGES, etc.) in the file. This re-export pattern means all existing `from app.core.constants import Region` imports throughout the app continue to work without changes.

4. **Update** `apps/efofx-estimate/app/utils/crypto.py`: Replace the file contents with a re-export from the shared package:
   ```python

   ```
   This preserves backward compatibility — all existing `from app.utils.crypto import ...` imports work.

5. **Update** `packages/efofx-shared/efofx_shared/__init__.py` to export version and key modules.

6. **Update** init files in `core/` and `utils/` subdirectories to export their public symbols.

Verify the exact uv workspace dependency syntax by checking uv docs if the manual approach fails.
