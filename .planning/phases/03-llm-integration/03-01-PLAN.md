<objective>
Harden the LLM service for per-request BYOK key injection, structured error handling, and response caching. Remove the settings.OPENAI_API_KEY fallback from all production code paths. Refactor ChatService and EstimationService to accept LLMService via dependency injection instead of creating their own instances.

Purpose: Phase 2 wired LLMService to accept an api_key parameter but kept a settings fallback for dev/testing. This plan removes that fallback from production paths, adds a FastAPI dependency that decrypts BYOK keys per-request, classifies OpenAI errors into actionable HTTP responses, and adds content-hash caching to avoid redundant API calls. This is the foundation every subsequent Phase 3 plan builds on.

Output: A production-ready LLMService with BYOK-only instantiation, error classification, response caching, and updated service/route wiring.
</objective>

2. **Add error classification function** (module-level, not method):
   ```python
   from openai import (
       AuthenticationError,
       RateLimitError,
       APITimeoutError,
       APIConnectionError,
       OpenAIError,
   )

   def classify_openai_error(exc: OpenAIError) -> tuple[str, int]:
       """Classify an OpenAI exception into (error_type, http_status).

       Returns:
           ("invalid_key", 402) — key is invalid or expired
           ("quota_exhausted", 402) — OpenAI quota used up
           ("transient", 503) — timeout/connection/rate-limit, retry eligible
           ("unknown", 500) — unexpected error
       """
       if isinstance(exc, AuthenticationError):
           return ("invalid_key", 402)
       if isinstance(exc, RateLimitError):
           if "insufficient_quota" in str(exc):
               return ("quota_exhausted", 402)
           return ("transient", 503)
       if isinstance(exc, (APITimeoutError, APIConnectionError)):
           return ("transient", 503)
       return ("unknown", 500)
   ```

3. **Add response caching** — add a module-level cache and helper:
   ```python
   import hashlib
   import json

   _response_cache: dict[str, str] = {}  # In-memory; upgrade to Valkey for multi-instance

   def _make_cache_key(messages: list[dict], model: str) -> str:
       """SHA-256 hash of (messages + model) for deterministic cache key."""
       payload = {"messages": messages, "model": model}
       serialized = json.dumps(payload, sort_keys=True, ensure_ascii=True)
       return hashlib.sha256(serialized.encode("utf-8")).hexdigest()
   ```

4. **Add cached wrapper to `generate_estimation`:**
   Before calling `self.client.beta.chat.completions.parse(...)`, compute cache key from the messages list and model. If cache hit, deserialize and return the cached `EstimationOutput`. On cache miss, call OpenAI, cache the result (as JSON string via `result.model_dump_json()`), and return.

   Update `generate_estimation` to accept an optional `use_cache: bool = True` parameter. When `use_cache=True` (default), check cache first. When `False`, bypass cache (useful for forced refresh).

5. **Add `get_llm_service` FastAPI dependency:**
   ```python
   from fastapi import Depends, HTTPException
   from app.core.security import get_current_tenant
   from app.services.byok_service import decrypt_tenant_openai_key
   from app.models.tenant import Tenant

   async def get_llm_service(tenant: Tenant = Depends(get_current_tenant)) -> LLMService:
       """FastAPI dependency: decrypt BYOK key and return scoped LLMService.
       Raises HTTP 402 if tenant has no stored OpenAI key.
       """
       api_key = await decrypt_tenant_openai_key(tenant.tenant_id)
       return LLMService(api_key=api_key)
   ```

6. **Add a `stream_chat_completion` async generator method** on LLMService for use by plan 03-04:
   ```python
   async def stream_chat_completion(self, messages: list[dict], temperature: float | None = None):
       """Async generator yielding content strings from a streaming chat completion.
       Filters out None chunks (role-only and finish_reason-only).
       Caller wraps in SSE formatting.
       """
       stream = await self.client.chat.completions.create(
           model=self.model,
           messages=messages,
           stream=True,
           temperature=temperature or self.temperature,
       )
       async for chunk in stream:
           content = chunk.choices[0].delta.content
           if content is not None:
               yield content
   ```
