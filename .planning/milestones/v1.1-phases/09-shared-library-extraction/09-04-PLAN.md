**Sections to include:**

1. **Code Style**
   - Python: black (line-length 88), flake8, mypy strict mode — reference existing pyproject.toml [tool.*] configs
   - TypeScript: ESLint + strict TypeScript — reference existing tsconfig strictness
   - Naming: snake_case for Python, camelCase for TypeScript, PascalCase for components/classes

2. **File & Folder Structure**
   - Python apps: `app/{api,core,models,services,utils}/` structure
   - React apps: `src/{components,hooks,types,lib}/` structure
   - Shared packages: mirror the relevant app structure
   - One component per file, co-located CSS modules

3. **Testing Patterns**
   - Python: pytest with asyncio_mode="auto", fixtures in conftest.py, test files mirror source structure
   - Frontend: (document current state — if no tests exist, note it as a gap)
   - Isolation tests for shared packages
   - Test naming: `test_{what}_{condition}_{expected}`

4. **Documentation Standards**
   - Python: docstrings on all public functions (Google style)
   - TypeScript: JSDoc on exported functions, interface comments for non-obvious fields
   - No Storybook (per REQUIREMENTS.md out-of-scope)

5. **Error Handling Patterns**
   - Python: specific exception types, no bare `except:`
   - API errors: structured `{error: string}` responses with appropriate HTTP status
   - Frontend: ErrorBoundary at layout level, toast for transient errors

6. **Logging Conventions**
   - Python: structured logging via `logging` stdlib
   - Log levels: DEBUG for development detail, INFO for operations, WARNING for degraded state, ERROR for failures
   - Include tenant_id in log context where applicable

7. **Import Conventions**
   - Shared package imports: `from efofx_shared.utils.crypto import ...` (not `from packages.efofx_shared...`)
   - Frontend shared imports: `import { Component } from '@efofx/ui'`
   - No circular imports between packages
