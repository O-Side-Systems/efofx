**Before moving anything:** Read each source component to understand its imports, props interface, and CSS class references. Map all CSS class names used.

1. **Create types file** `packages/efofx-ui/src/types/estimation.ts`:
   - Read `apps/efofx-widget/src/types/widget.d.ts` to find the `EstimationOutput` interface
   - Create a standalone copy in the UI package — do NOT import from the widget app
   - Export the interface so EstimateCard can import it locally

2. **Extract each component** into its own directory under `packages/efofx-ui/src/components/`:

   For each component (ChatBubble, EstimateCard, TypingIndicator, ErrorBoundary, LoadingSkeleton):
   a. Read the source file
   b. Create the component file in `packages/efofx-ui/src/components/{Name}/{Name}.tsx`
   c. Strip any widget-specific or app-specific imports
   d. For EstimateCard: import EstimationOutput from `../../types/estimation`
   e. Export the component and its props type as named exports

3. **Create CSS modules** for each component that uses CSS classes:
   - Read `apps/efofx-widget/src/widget.css` to find the CSS rules for each component's class names
   - Create `.module.css` files with camelCase class names:
     - `efofx-bubble-wrapper` → `bubbleWrapper`
     - `efofx-bubble-wrapper--user` → `bubbleWrapperUser`
     - `efofx-bubble-wrapper--assistant` → `bubbleWrapperAssistant`
     - `efofx-bubble` → `bubble`
     - etc. (map all class names from the source CSS)
   - Update component TSX to `import styles from './ComponentName.module.css'` and use `styles.camelCaseName`
   - Preserve CSS custom property hooks (var(--brand-primary), var(--brand-accent), etc.) for theming

4. **ErrorBoundary** likely has no CSS — if so, skip the .module.css file.

5. **Update barrel export** `packages/efofx-ui/src/index.ts`:
   ```typescript
   export { ChatBubble } from './components/ChatBubble/ChatBubble';
   export type { ChatBubbleProps } from './components/ChatBubble/ChatBubble';
   export { EstimateCard } from './components/EstimateCard/EstimateCard';
   export type { EstimateCardProps, EstimationOutput } from './components/EstimateCard/EstimateCard';
   export { TypingIndicator } from './components/TypingIndicator/TypingIndicator';
   export { ErrorBoundary } from './components/ErrorBoundary/ErrorBoundary';
   export { LoadingSkeleton } from './components/LoadingSkeleton/LoadingSkeleton';
   ```
   Adjust export names based on what each component actually exports.

IMPORTANT CSS MODULE PITFALL: Do NOT use hyphenated names as JS property access (`styles.efofx-bubble`). All CSS class names MUST be camelCase in the .module.css files.
  </action>
