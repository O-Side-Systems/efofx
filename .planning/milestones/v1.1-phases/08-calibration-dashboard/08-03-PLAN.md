**Dashboard.tsx structure:**

```tsx
// State
const [dateRange, setDateRange] = useState('all');
const { data, isPending, isError, error, refetch } = useCalibration(dateRange);
```

**Layout (single scrollable page per user decision):**

```
Header: "Calibration Dashboard" + DateRangeFilter (right-aligned)
─────────────────────────────────────────────────

if isPending → <LoadingSkeleton />

if isError → Error card with message + "Retry" button (calls refetch())

if data.below_threshold → <ThresholdProgress outcomeCount={data.outcome_count} threshold={data.threshold} />

if !data.below_threshold →
  <CalibrationMetrics metrics={data} />
  <section class="section card">
    <h2>Accuracy Distribution</h2>
    <AccuracyBucketBar buckets={data.accuracy_buckets} />
  </section>
  <section class="section card">
    <AccuracyTrendLine />
  </section>
  <section class="section card">
    <h2>Reference Class Breakdown</h2>
    <ReferenceClassTable referenceClasses={data.by_reference_class} />
  </section>
```

**Key implementation details:**
- Use `isPending` not `isLoading` (React Query v5)
- DateRangeFilter `onChange` updates the `dateRange` state — React Query automatically refetches because the queryKey changes (`['calibration', dateRange]`)
- Error state: show `error.message` or a generic "Failed to load calibration data" in a card with a blue "Retry" button
- When `below_threshold` is true, do NOT show any chart components, date range filter, or partial metrics — only the ThresholdProgress component (per user decision: "No partial accuracy metrics are shown")
- The DateRangeFilter should still be visible in the header even below threshold (the dashboard link is always visible per user decision), but when below_threshold is true, changing the range will just refetch and still show below-threshold state
- Page heading uses semantic `<h1>` with `font-size: 1.5rem; font-weight: 600`
