**Part B: Refactor LLMService to accept per-request API key (Gap 3)**

Modify `apps/efofx-estimate/app/services/llm_service.py`:

1. **Change __init__ signature** to accept an optional `api_key` parameter:
```python
def __init__(self, api_key: Optional[str] = None) -> None:
    self.api_key = api_key or settings.OPENAI_API_KEY
    self.client = AsyncOpenAI(api_key=self.api_key)
    self.model = settings.OPENAI_MODEL
    self.max_tokens = settings.OPENAI_MAX_TOKENS
    self.temperature = settings.OPENAI_TEMPERATURE
```

The `api_key` parameter is the per-request decrypted BYOK key. The fallback to `settings.OPENAI_API_KEY` preserves backward compatibility for dev/testing and for the transition period until Phase 3 (LLM-01) wires the full per-request flow.

2. **Design note in docstring:** Add a class-level docstring explaining:
- In production, callers MUST provide `api_key` (the decrypted BYOK key from `decrypt_tenant_openai_key()`)
- The `settings.OPENAI_API_KEY` fallback exists only for dev/testing and will be removed when Phase 3 implements per-request client injection (LLM-01)
- The intent is that each LLM call uses the tenant's own key, never the platform key in production

3. **Do NOT change the method signatures** of generate_response, classify_project, or generate_estimation — they remain unchanged. The key injection happens at construction time, and Phase 3 (LLM-01) will handle per-request instantiation.

4. **Do NOT import or call decrypt_tenant_openai_key here** — that wiring belongs in Phase 3 (LLM-01) where the API endpoints create LLMService instances per-request. This plan only makes LLMService READY to receive a BYOK key.

The pattern for Phase 3 callers will be:
```python
key = await decrypt_tenant_openai_key(tenant_id)  # 402 if missing
llm = LLMService(api_key=key)
result = await llm.generate_estimation(...)
```
