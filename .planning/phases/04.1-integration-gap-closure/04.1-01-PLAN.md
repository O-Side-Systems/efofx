**INT-02: Wire create_indexes() to app startup in main.py**

1. Update the import line in `main.py` (line 22):
```python
from app.db.mongodb import connect_to_mongo, close_mongo_connection, health_check as db_health_check, create_indexes
```

2. Restructure the lifespan function so `connect_to_mongo()` and `create_indexes()` are NOT inside the swallowing try/except. Per user decision, both must fail loudly (crash the app). Model after the PromptService.load_all() pattern. The new lifespan should be:

```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    """Application lifespan events."""
    # Startup
    logger.info("Starting efOfX Estimation Service...")
    await connect_to_mongo()
    logger.info("MongoDB connection established")
    await create_indexes()
    logger.info("Database indexes created")

    # Load versioned prompts -- critical dependency, fail startup if missing
    prompts_dir = os.path.join(
        os.path.dirname(os.path.dirname(os.path.abspath(__file__))),
        "config",
        "prompts",
    )
    try:
        PromptService.load_all(prompts_dir)
        logger.info("Prompt registry loaded")
    except Exception as e:
        logger.error(f"Failed to load prompt registry: {e}")
        raise  # Prompts are critical -- fail startup if they can't load

    yield

    # Shutdown
    logger.info("Shutting down efOfX Estimation Service...")
    await close_mongo_connection()
    logger.info("MongoDB connection closed")
```

Key changes:
- `connect_to_mongo()` moved OUTSIDE the try/except (it was swallowing errors before)
- `create_indexes()` called immediately after connect, also outside try/except
- Both propagate exceptions naturally (app won't start)
- PromptService block stays the same (already had raise)

3. Add per-index logging to `create_indexes()` in `mongodb.py`. After each `create_index` call, add a `logger.info()` line. For example:
```python
await db["tenants"].create_index("email", unique=True)
logger.info("Index confirmed: tenants.email_1")
await db["tenants"].create_index("tenant_id", unique=True)
logger.info("Index confirmed: tenants.tenant_id_1")
```

Add similar `logger.info(f"Index confirmed: {collection_name}.{index_name}")` after EVERY `create_index` call in the function. Use MongoDB's standard index naming convention (field names joined by underscores with direction suffixes).

The final summary log "Database indexes created successfully" can remain.
